An **algorithm** is a **finite, step-by-step procedure** for solving a problem or performing a task.

- **Well-defined** â†’ every step is clear, unambiguous.
- **Finite** â†’ must end in a limited number of steps.

## ğŸ”§ 3. Why Do We Use Flowcharts?

Flowcharts are **visual representations** of algorithms.

### Benefits:
- easier to read and understand,
- ideal for beginners,
- helps in debugging logical mistakes,
- provides a clear overview of control flow,
- useful before writing actual C++ code.

Flowcharts use standard symbols:
- oval â†’ start/end  
- parallelogram â†’ input/output  
- rectangle â†’ process  
- diamond â†’ decision
